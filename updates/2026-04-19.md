# Supervisor-Facing Update: Sequential NIH -> CheXpert -> MIMIC Warm-Start Forgetting Study

Prepared on April 19, 2026 (UTC).

## Objective

After the earlier `NIH -> CheXpert` warm-start experiments, the next concrete question was what happens when the same linear head is adapted sequentially across domains rather than evaluated only once after a single target adaptation step.

The specific study was:

1. train a source head on `NIH`
2. test direct transfer from that `NIH` head to both `CheXpert` and `MIMIC`
3. warm-start the same head on `CheXpert` and measure both `CheXpert` performance and `NIH` forgetting
4. warm-start that `CheXpert`-adapted head on `MIMIC` and measure `MIMIC` performance plus retention on both earlier domains

The goal was to quantify both transfer and forgetting in a simple head-only sequential adaptation chain.

## Experimental protocol

- Date:
  - `April 19, 2026 (UTC)`
- Backbone and features:
  - frozen `CXR Foundation`
  - `general` embeddings
  - `avg` token pooling
  - `768`-dimensional features
- Head:
  - multilabel linear probe
- Model size and compute for the trainable stage:
  - linear head only: `5.4k` parameters
  - linear head only: about `10.8k` FLOPs per image for the `768 -> 7` classifier
  - note: the frozen `CXR Foundation` backbone was used through pre-exported embeddings, so full backbone FLOPs were outside this head-training stage and were not logged by the workspace runs
- Optimization:
  - `AdamW`
  - learning rate `1e-3`
  - weight decay `1e-4`
  - batch size `512`
  - early stopping with patience `5`
- Data scale:
  - same `pilot5h` common `7`-label setup already used elsewhere in the workspace
- Split sizes used:
  - `NIH`: `10,000 train / 1,000 val / 2,000 test`
  - `CheXpert`: `1,000 train / 1,000 val / 234 test`
  - `MIMIC`: intended `1,000 train / 1,000 val / 676 test`
- Important MIMIC detail:
  - the exported `MIMIC train` embedding set contained `998` usable rows rather than `1000`
  - the missing `2` rows were the same corrupt JPGs identified earlier during the `MIMIC` embedding export
- Manifest handling:
  - `NIH` and `CheXpert` rows came from the existing `pilot5h` manifest
  - `MIMIC` rows were replaced with the binary-normalized target manifest so that uncertain `-1` labels were mapped to `0` before `BCEWithLogitsLoss`
- Sequential stages:
  - Stage A: train on `NIH`, select on `NIH val`, evaluate on `NIH test`, `CheXpert test`, and `MIMIC test`
  - Stage B: initialize from Stage A, train on `CheXpert`, select on `CheXpert val`, evaluate on `CheXpert test` and `NIH test`
  - Stage C: initialize from Stage B, train on `MIMIC`, select on `MIMIC val`, evaluate on `MIMIC test`, `CheXpert test`, and `NIH test`

| Stage | Best epoch | NIH test AUROC | CheXpert test AUROC | MIMIC test AUROC |
| --- | ---: | ---: | ---: | ---: |
| Stage A: `NIH` source head | `24` | `0.8461` | `0.8486` | `0.7370` |

| Quantity | Delta AUROC |

## Interpretation

The most important result is that the simple sequential warm-start chain does **not** show a uniform “adaptation helps every downstream domain” pattern.

Instead, the pattern is asymmetric:

- the `NIH` source head already transfers very strongly to `CheXpert` zero-shot, reaching `0.8486` AUROC
- after explicit `CheXpert` adaptation, `CheXpert test` AUROC drops to `0.7882`
- after the later `MIMIC` adaptation step, `CheXpert` partially recovers to `0.8135`, but still remains below the original `NIH` zero-shot `CheXpert` result

So in this pilot setting, the first `NIH -> CheXpert` adaptation step should **not** be interpreted as a successful improvement over the source model on `CheXpert`. The defensible reading is the opposite: the source model already generalized unusually well to this `CheXpert` holdout, and the plain warm-start adaptation step degraded that performance.

The forgetting story is also more specific than a simple catastrophic-collapse narrative:

- the main `NIH` forgetting event happens at the `CheXpert` adaptation step, where `NIH test` AUROC falls by about `-0.0251`
- the later `MIMIC` adaptation step does **not** introduce additional meaningful `NIH` damage
- in fact, `NIH` recovers slightly from `0.8210` to `0.8221` after the `MIMIC` step

The `MIMIC` transfer result is the cleanest positive finding in the study:

- the original `NIH` source head reaches `0.7370` AUROC on `MIMIC test`
- the final `MIMIC`-adapted head improves that to `0.7770`
- this is a gain of about `+0.0400` while keeping final `NIH` retention roughly flat relative to Stage B

So the strongest conclusion from this run family is:

- `MIMIC` adaptation helps `MIMIC`
- the main forgetting cost is paid during the earlier `CheXpert` adaptation step
- the `CheXpert` target-adaptation recipe used here is weaker than the already-strong `NIH` zero-shot transfer baseline on this pilot holdout

## Test-set correctness and caveats

The test-split concern was checked explicitly for this run family.

- Stage A used `d1_chexpert/test` with `234` rows and `d2_mimic/test` with `676` rows
- Stage B used `d1_chexpert/test` with `234` rows
- Stage C used `d1_chexpert/test` with `234` rows and `d2_mimic/test` with `676` rows
- the merged manifest had `0` `row_id` overlap across `train`, `val`, and `test` inside each dataset
- the `998`-row `MIMIC train` count is due to skipped corrupt images, not to an incorrect split definition

There are still two caveats worth stating clearly:

1. this is a `pilot5h` subset study, not a full-dataset result
2. the current run family does not include the Stage B `CheXpert`-adapted checkpoint evaluated on `MIMIC`, so the full `3 x 3` stage-by-domain matrix still has one missing cell

## Supervisor-facing message

The supervisor-facing conclusion is:

1. The simple sequential warm-start recipe does **not** deliver monotonic cross-domain improvement.
2. On this pilot setup, the original `NIH` head is already very strong on `CheXpert`, and the plain `CheXpert` adaptation step actually makes `CheXpert` worse.
3. The main measurable source forgetting occurs during `NIH -> CheXpert` adaptation, not during the later `CheXpert -> MIMIC` step.
4. The final `MIMIC` adaptation is still useful because it improves `MIMIC` test AUROC by about `+0.0400` over the original `NIH` zero-shot baseline while keeping `NIH` roughly stable relative to Stage B.
5. The next clean follow-up would be either:
   - backfill the missing Stage B `MIMIC` evaluation to complete the matrix, or
   - test a stronger `CheXpert` adaptation rule such as `LwF` in the same sequential chain, since the plain warm-start rule is clearly not sufficient here.
