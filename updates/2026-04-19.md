# Supervisor-Facing Update: Oracle Baselines, NIH Transfer, and Naive Continual Learning

Prepared on April 19, 2026 (UTC).

## Objective

This update reorganizes the current `CXR Foundation` pilot results into the three comparisons that matter most for the next supervisor-facing report:

1. independent single-domain oracle baselines
2. cross-domain transfer from `NIH` with no adaptation
3. the naive continual-learning baseline `NIH -> CheXpert -> MIMIC`

The purpose is to separate three different questions that had started to blur together:

- what is the per-domain ceiling of the current embedding space under simple linear probing?
- how much transfer do we get for free from `NIH` before any domain adaptation?
- how much forgetting does the naive sequential fine-tuning recipe introduce?

## Shared setup

- Date:
  - `April 19, 2026 (UTC)`
- Backbone and features:
  - frozen `CXR Foundation`
  - `general` embeddings
  - `avg` token pooling
  - `768`-dimensional features
- Head:
  - multilabel linear probe
- Model size and compute for the trainable stage:
  - linear head only: `5.4k` parameters
  - linear head only: about `10.8k` FLOPs per image for the `768 -> 7` classifier
  - note: full `CXR Foundation` backbone FLOPs were not logged in these runs because all experiments used pre-exported embeddings
- Optimization:
  - `AdamW`
  - learning rate `1e-3`
  - weight decay `1e-4`
  - batch size `512`
  - early stopping patience `5`
- Label space:
  - common `7` labels
- Data scale:
  - `NIH`: `10,000 train / 1,000 val / 2,000 test`
  - `CheXpert`: `1,000 train / 1,000 val / 234 test`
  - `MIMIC`: `1,000 train / 1,000 val / 676 test`, with `998` embedded train rows because `2` corrupt JPGs were skipped
- Manifest handling:
  - `MIMIC` labels were binary-normalized so uncertain `-1` values were mapped to `0` before `BCEWithLogitsLoss`

## 1. Independent single-domain oracle baselines

These runs answer the simplest possible question: if each domain is allowed to train its own linear probe on the same frozen embedding space, how high can the same-domain test AUROC go?

### Results

| Training rule | Run | Best epoch | Same-domain test AUROC | Same-domain test AP | Params | FLOPs / image |
| --- | --- | ---: | ---: | ---: | ---: | ---: |
| `NIH` only | `exp0001` | `24` | `0.8461` on `NIH test` | `0.2545` | `5.4k` | `10.8k` |
| `CheXpert` only | `exp0004` | `50` | `0.7703` on `CheXpert test` | `0.4714` | `5.4k` | `10.8k` |
| `MIMIC` only | `exp0005` | `24` | `0.7619` on `MIMIC test` | `0.3970` | `5.4k` | `10.8k` |

### Interpretation

These are the current per-domain oracle ceilings for the present embedding space under the simplest head-only recipe.

The resulting ranking is:

- strongest same-domain AUROC: `NIH` at `0.8461`
- second: `CheXpert` at `0.7703`
- third: `MIMIC` at `0.7619`

So if later adaptation methods are discussed as “good” or “bad,” they should be interpreted relative to these domain-specific ceilings rather than in isolation.

## 2. Cross-domain transfer baseline

This baseline trains only on `NIH` and then tests directly on `CheXpert` and `MIMIC` with no adaptation at all.

### Results

| Train domain | Test domain | Run | Test AUROC | Test AP |
| --- | --- | --- | ---: | ---: |
| `NIH` | `NIH` | `exp0001` | `0.8461` | `0.2545` |
| `NIH` | `CheXpert` | `exp0001` | `0.8486` | `0.5470` |
| `NIH` | `MIMIC` | `exp0001` | `0.7370` | `0.3760` |

### Comparison to the oracle baselines

| Comparison | Delta AUROC |
| --- | ---: |
| `NIH -> CheXpert` transfer minus `CheXpert` oracle | `+0.0783` |
| `NIH -> MIMIC` transfer minus `MIMIC` oracle | `-0.0249` |

### Interpretation

This transfer baseline does **not** show the usual simple “off-domain is always worse than in-domain” story.

Instead:

- direct `NIH -> CheXpert` transfer is unusually strong and actually exceeds the `CheXpert`-only oracle by about `+0.0783`
- direct `NIH -> MIMIC` transfer is weaker than the `MIMIC` oracle by about `-0.0249`

So the present raw domain gap is asymmetric:

- `CheXpert` already looks highly aligned with the `NIH` source head in this pilot setting
- `MIMIC` still shows a clear remaining transfer gap that target adaptation could plausibly close

## 3. Naive continual baseline

This is the forgetting baseline that later continual-learning methods will need to beat.

The recipe is intentionally simple:

1. train a source head on `NIH`
2. fine-tune that head on the `CheXpert` subset
3. fine-tune that `CheXpert`-adapted head on the `MIMIC` subset

### Results

| Stage | Run | Best epoch | NIH test AUROC | CheXpert test AUROC | MIMIC test AUROC |
| --- | --- | ---: | ---: | ---: | ---: |
| Stage A: `NIH` source head | `exp0001` | `24` | `0.8461` | `0.8486` | `0.7370` |
| Stage B: after `CheXpert` fine-tune | `exp0002` | `24` | `0.8210` | `0.7882` | not evaluated in this run |
| Stage C: after `MIMIC` fine-tune | `exp0003` | `6` | `0.8221` | `0.8135` | `0.7770` |

### Key deltas

| Quantity | Delta AUROC |
| --- | ---: |
| `NIH` after `CheXpert` fine-tune vs Stage A | `-0.0251` |
| `NIH` after final `MIMIC` fine-tune vs Stage A | `-0.0240` |
| `CheXpert` after Stage B vs Stage A zero-shot transfer | `-0.0604` |
| `CheXpert` after Stage C vs Stage A zero-shot transfer | `-0.0351` |
| `CheXpert` after Stage C vs Stage B | `+0.0253` |
| `MIMIC` after Stage C vs Stage A zero-shot transfer | `+0.0400` |
| `MIMIC` after Stage C vs `MIMIC` oracle | `+0.0151` |

### Interpretation

This naive continual-learning chain establishes a clear forgetting baseline.

The main pattern is:

- the major forgetting event happens during the `NIH -> CheXpert` step
- `NIH` falls from `0.8461` to `0.8210`, a drop of about `-0.0251`
- the later `CheXpert -> MIMIC` step does **not** create another large `NIH` collapse
- `NIH` actually stabilizes slightly, moving from `0.8210` to `0.8221`

The target-domain story is mixed rather than uniformly positive:

- `CheXpert` gets worse after explicit `CheXpert` fine-tuning when compared against the original `NIH` zero-shot transfer baseline
- `CheXpert` falls from `0.8486` to `0.7882` at Stage B
- after the later `MIMIC` step, `CheXpert` partially recovers to `0.8135`, but it still remains below the original `NIH` zero-shot level

By contrast, `MIMIC` is the clean positive case:

- Stage A zero-shot `NIH -> MIMIC` gives `0.7370`
- Stage C improves this to `0.7770`
- that is a gain of about `+0.0400`
- the final `MIMIC` result also slightly exceeds the independent `MIMIC` oracle by about `+0.0151`

So the naive continual baseline is not “catastrophic forgetting everywhere.” The better reading is:

- `CheXpert` adaptation is the step that causes most of the measurable source forgetting
- `MIMIC` adaptation is useful for `MIMIC`
- the current plain warm-start `CheXpert` step is weaker than the already-strong `NIH` zero-shot baseline on this pilot holdout

## Test-set correctness and caveats

The split concern was checked explicitly for this run family.

- `NIH -> CheXpert` transfer used the actual `CheXpert test` split with `234` rows
- `NIH -> MIMIC` transfer used the actual `MIMIC test` split with `676` rows
- the merged manifest had `0` `row_id` overlap across `train`, `val`, and `test` inside each dataset
- `MIMIC train` used `998` embedded rows because `2` images were corrupt and skipped at export time

Two caveats still matter:

1. this remains a `pilot5h` subset study rather than a full-dataset result
2. Stage B was not evaluated on `MIMIC`, so the full `3 x 3` stage-by-domain matrix still has one missing cell

## Supervisor-facing message

The clearest supervisor-facing conclusion is:

1. The single-domain oracle ceilings for the current embedding space are `0.8461` on `NIH`, `0.7703` on `CheXpert`, and `0.7619` on `MIMIC`.
2. Raw `NIH -> CheXpert` transfer is already stronger than the `CheXpert`-only oracle on this pilot holdout, which means the simple target-only `CheXpert` head is **not** the right baseline to beat the source model.
3. Raw `NIH -> MIMIC` transfer is still below the `MIMIC` oracle, so `MIMIC` remains the clearer target for useful adaptation.
4. The naive continual baseline shows that most measurable forgetting happens during the `NIH -> CheXpert` step.
5. The `CheXpert -> MIMIC` step improves `MIMIC` substantially and does not add much further `NIH` damage.
6. Any continual-learning method we test next should therefore be judged against this naive baseline on two axes:
   - can it retain `NIH` better than the current `-0.0251` drop after `CheXpert` fine-tuning?
   - can it preserve or improve the `MIMIC` gain without letting the `CheXpert` step remain weaker than the original `NIH` zero-shot transfer?
