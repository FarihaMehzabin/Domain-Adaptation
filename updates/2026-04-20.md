## 2. Architecture, parameters, FLOPs, and memory

### Core source model used throughout

- Model: frozen linear multilabel head
- Shape: `768 -> 7`
- Trainable parameters when trained directly: `5,383`
- Forward FLOPs per image: `10,759`

This linear head is the model class used for:

- `stage0_source`
- `source_only`
- `finetune_seq`
- `lwf`
- `l2_anchor`
- `ewc`
- `vq_summary_replay`

### Prototype memory used by retrieval-based methods

- Prototype construction: `4` positive + `2` negative state prototypes per label
- Labels: `7`
- Total prototypes in the saved `NIH` source bank: `42`
- Saved memory size: `131,460` bytes = `0.12537 MB`

### Main method family

The `main_method` is:

1. a frozen previous-stage linear head
2. a prototype-retrieval prior from the saved source bank
3. a residual logit adapter
4. a scalar gate that decides how much residual to apply

The key dimensions are:

- Residual adapter input: `768 + 7 + 7 = 782`
- Residual adapter MLP: `782 -> 128 -> 7`
- Gate input: `768 + 7 + 7 + 2 = 784`
- Gate MLP: `784 -> 64 -> 1`

Cost for the Stage-1 `main_method` / `harder_gate_clipping` family:

- Trainable parameters: `151,432`
- Frozen parameters carried from previous linear head: `5,383`
- Residual-adapter FLOPs: `202,119`
- Gate FLOPs: `100,545`
- Retrieval FLOPs: `70,698`
- Adaptation overhead per image: `373,362`
- Total forward cost including the frozen previous head: about `384,121` FLOPs/image

### Other important method variants

- `fixed_alpha_mix`: no extra learned parameters at evaluation time; mixes source probabilities with prototype prior using `alpha` chosen from `{0.1, 0.25, 0.5}`.
- `smaller_bottleneck`: same `main_method` design but `bottleneck_dim=32`; `75,592` trainable parameters.
- `tiny_logit_correction`: per-label affine correction on frozen logits; `14` trainable parameters and `21` extra FLOPs/image on top of the frozen linear head.
- `labelwise_trust_region`: per-label disagreement/uncertainty-gated residual correction with prototype replay, gate sparsity, margin preservation, and trust-region penalties.

### Main training hyperparameters

- Stage 0 real run:
  - `epochs=20`, `batch_size=256`, `lr=1e-3`, `weight_decay=1e-4`, `patience=5`, `fisher_max_batches=25`
- Stage 1 linear baselines:
  - `epochs=20`, `batch_size=256`, `lr=5e-4`, `weight_decay=1e-4`, `patience=5`
  - `LwF`: `distill_weight=1.0`, `temperature=2.0`
  - `L2-anchor`: `l2_anchor_weight=1e-4`
  - `EWC`: `ewc_weight=1.0`
  - `VQ summary replay`: `replay_weight=1.0`
- Stage 1 `main_method`:
  - `epochs=20`, `batch_size=256`, `lr=5e-4`, `weight_decay=1e-4`, `patience=5`
  - `distill_weight=1.0`, `distill_temperature=2.0`
  - `prototype_replay_weight=1.0`, `residual_zero_weight=0.1`
  - `top_k=8`, `temperature=0.1`
  - `gate_hidden_dim=64`, `bottleneck_dim=128`, `dropout=0.1`
- `harder_gate_clipping` rescue adds:
  - `old_like_similarity_threshold=0.7`
  - `old_like_gate_cap=0.1`

## 5. The clean explanation to give your supervisor

If you need the shortest accurate talk track, use this:

1. `paper_v1` is a clean frozen-embedding continual-learning package built around a `768 -> 7` chest-X-ray head and a compact retrieval-based adaptation model.
2. The first real blocker was data, not modeling: `CheXpert` had to be refreshed to become patient-disjoint, and `MIMIC` train assets are still incomplete.
3. Once `CheXpert` was refreshed, the frozen `NIH` source head already transferred strongly to `CheXpert` at about `0.85` macro AUROC, so the real Stage-1 problem became retention rather than raw transfer.
4. The original `main_method` was not good enough to promote because it improved `CheXpert` but forgot too much `NIH`.
5. Postmortem showed the gate was effectively fully open almost everywhere, so the model was over-correcting even on old-like samples.
6. A simple rescue, `harder_gate_clipping`, fixed that behavior and remained the best seen-average Stage-1 rescue in the saved runs.
7. Later noon multiseed reruns added `labelwise_trust_region` and richer diagnostics, but the final saved promotion artifact at `12:21 UTC` still says **do not promote any Stage-1 method to Stage 2 yet**.
8. Stage 2 on `MIMIC` is not yet done in `paper_v1`; only repair preparation exists today, so that part should be presented as pending.
